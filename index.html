Simple

Lexicographic

Rank alternatives by the single most important attribute.

Learn more

How it works

  • Rank the criteria by weight (importance).
  • The criterion with the highest weight is the deciding factor.
  • Alternatives are ranked based solely on their score in that top criterion.

Strengths & Limitations

Very simple and fast, but ignores all criteria except the most important one. Best used as a quick initial filter.

✓
Simple

Pugh Method

Compare against a baseline using +, S, − scoring.

Learn more

How it works

  • Choose one alternative as the baseline.
  • For every other alternative, compare each criterion: better (+), same (S), or worse (−) vs. baseline.
  • Count the total +, S, and − marks. More + and fewer − is better.

Strengths & Limitations

Easy to understand and doesn't require precise scoring. However, it doesn't account for the magnitude of differences or weights directly in the comparison.

✓
Moderate

QFD-like

Score with 0, 1, 3, 9 goodness scale. Weighted & unweighted sums.

Learn more

How it works

  • Rank criteria by system objectives and assign weights.
  • Rate each alternative on each criterion: 0 (no relation), 1 (weak), 3 (moderate), 9 (strong).
  • Calculate unweighted sum and weighted sum across all criteria.
  • Select the alternative with the highest scores.

Strengths & Limitations

The 0-1-3-9 scale forces clear differentiation. Good for early concept screening. Scores are subjective and don't use the actual natural units of the criteria.

✓
Moderate

Modified Decision Matrix

Normalize scores to [0,1] and compute weighted sums.

Learn more

How it works

  • Define min and max desirable values for each attribute.
  • Normalize each value to a 0–1 scale:
    • If maximizing: (Value − Min) / (Max − Min)
    • If minimizing: (Value − Max) / (Min − Max)
  • Multiply each normalized score by its weight and sum across criteria.

Strengths & Limitations

Uses natural units and accounts for weights properly. Assumes a linear relationship between attribute value and desirability.

✓
Advanced

Multi-Attribute Utility (MAU)

Define utility curves per attribute, interpolate values, weighted sum.

Learn more

How it works

  • For each criterion, define a Single Attribute Utility (SAU) curve mapping raw values to a 0–1 utility scale.
  • Curves can be linear, polynomial, exponential, or S-shaped to capture non-linear preferences.
  • Look up the utility score for each alternative's attribute value on the curve.
  • Compute MAU = weighted sum of all SAU scores: U = Σ ki · Ui

Strengths & Limitations

The most comprehensive method — captures nonlinear preferences and produces the most nuanced ranking. Requires more effort to define curves. Complexity increases in order: Lexicographic → Pugh → QFD → Modified DM → MAU.
